Versioning Specification

Release & CI Versioning

Format Architecture

  • Use 2026.m.m-build.${build number} for CI builds.
  • Use 2026.m.m-{iteration 1...n} for test versions (stable).
  • Use 2026.m.m for release versions.

Timeline Examples

Releases update iteratively per month. When approaching the end of the year, the year tag must be advanced:

  • 2026.1.0 → October 2026
  • 2026.1.1 → November 2026
  • 2026.1.2 → December 2026 (CRITICAL: In December, the project must use/prepare the New Year format for upcoming releases).

gradle.properties Standard

It must ALWAYS provide these 5 variables (values shown are examples):

# --- Project Identity ---
project-group=io.github.mcengine
git-org-name=MCEngine
git-repository=repo

# --- Versioning ---
project-version=2026.1.2
project-iteration=1
